Render Changelog API
====================

This tiny API is intended to be deployed on Render and uses Supabase (Postgres) as the backing store.

Environment variables (set on Render):
- `SUPABASE_URL` — your Supabase project URL (https://xyz.supabase.co)
- `SUPABASE_SERVICE_ROLE` — Supabase service_role key (kept secret on Render)
- `API_SECRET` — a simple shared secret that the admin panel will send in `x-api-secret` header to authorize writes

Database table (SQL)
--------------------
Run this in Supabase SQL editor to create the `changelog` table:

```sql
create table if not exists changelog (
  id bigint generated by default as identity primary key,
  title text,
  content text not null,
  date bigint not null
);
```

Deploy
------
1. Push this folder to a Git repo.
2. Create a new Web Service on Render pointing to the repo and folder `render-changelog-api`.
3. Set the environment variables described above in Render's dashboard.
4. Deploy. The service exposes `/changelog` (GET), `/changelog` (POST, requires `x-api-secret`), and `/changelog/:id` (DELETE, requires `x-api-secret`).

Client setup
------------
1. Set `window.CHANGELOG_API_BASE` to your deployed Render service root (e.g., `https://your-service.onrender.com`) before `app.js` runs. Example in `index.html`:

```html
<script>window.CHANGELOG_API_BASE = 'https://your-service.onrender.com';</script>
<script src="app.js"></script>
```

2. On the admin site, set `window.CHANGELOG_API_SECRET` (only on the server-rendered admin page) or ensure your admin modal sends `x-api-secret` with the secret. Prefer rendering the admin page server-side to avoid exposing the secret in client JS.
